# 山本さん YouTube台本生成スキル

テーマを受け取り、Xリサーチ（任意）→ 台本構成 → RAG検索 → YouTube台本生成 を行う。
各ステップでユーザーの承認を得てから次に進む。
承認は必ず **AskUserQuestionツール** を使い、選択肢付きで確認すること。

> **X記事スキルからの連携時:** yamamoto-x-postスキルで「YouTube台本も作る」を選択した場合、
> すでにテーマ・Xリサーチ結果・記事構成・RAGデータが会話コンテキストにある。
> その場合はステップ①〜④をスキップし、ステップ⑤に直行する。
> 「X記事作成で収集したリサーチ・RAGデータを引き継ぎます。YouTube台本生成に進みます。」と表示してからステップ⑤を開始すること。

## RAGデータ運用原則（厳守）

1. RAGデータは**広く取得**してよい（5〜10件読込はOK）
2. 反映する**コア具体事例は 2〜3件まで**（体験談1件＋Q&A 1〜2件が目安、Shorts各1件まで）
3. 1本の台本内で具体事例を過剰に並列せず、テーマ一貫性を最優先
4. 使用した事例は `_used/` へ移動し、`USAGE_LOG.md` にNo付きで記録
5. 直近30日 or 直近10投稿以内の事例は採用優先度を下げる（完全除外はしない）
6. 再利用時は以前と同じ切り口を避ける

詳細は `_shared/rag-usage-policy.md` を参照すること。

## ステップ② Xで最新情報をリサーチ

> Xリサーチ=OFFの場合はこのステップをスキップし、ステップ③も省略してステップ④へ進む。

**やること:**

3段階のフォールバック方式でXの最新情報を収集する。
各Phaseで十分な結果（具体的な投稿内容を含む3件以上）が得られたら、次のPhaseはスキップする。

### Phase 1: WebSearchで検索（無料）
1. WebSearchツールを使い、以下の検索クエリでXの最新投稿・トレンドを調査する:
   - `{テーマ} site:x.com`
   - `{テーマ} Twitter トレンド`
   - テーマに関連するキーワードの組み合わせ（2〜3パターン）
2. **判定:** 具体的な投稿内容を含む結果が3件以上 → Phase 2, 3をスキップ

### Phase 2: Nitterでフォールバック補完（無料）
3. Phase 1の結果が不十分な場合、Nitter経由で追加リサーチする:
   - WebFetchツールで `https://xcancel.com/search?q={テーマ}&f=tweets` を取得
   - 取得できない場合は `https://twiiit.com/search?q={テーマ}&f=tweets` を試す
   - それでも取得できない場合は `https://nitter.privacydev.net/search?q={テーマ}&f=tweets` を試す
4. **判定:** Phase 1 + 2 の合計で3件以上 → Phase 3をスキップ

### Phase 3: X API で最終補完（従量課金）
5. Phase 1 + 2 でも情報が不十分な場合、X APIで検索する:
   - 環境変数 `X_API_BEARER_TOKEN` が設定されているか確認
   - 未設定の場合: ユーザーに「X APIが未設定のため、現在の情報で進めます」と通知し、手持ちの情報で進める
   - 設定済みの場合: 以下のコマンドでツイート検索を実行
   ```bash
   curl -s -H "Authorization: Bearer ${X_API_BEARER_TOKEN}" \
     "https://api.x.com/2/tweets/search/recent?query={テーマ}&max_results=20&tweet.fields=created_at,public_metrics,text" \
     | python3 -c "
   import json, sys
   data = json.load(sys.stdin)
   for tweet in data.get('data', []):
       print(f\"[{tweet.get('created_at','')}] {tweet['text']}\")
       metrics = tweet.get('public_metrics', {})
       print(f\"  likes: {metrics.get('like_count',0)}, RT: {metrics.get('retweet_count',0)}\")
       print()
   "
   ```

6. 全Phaseの結果を統合し、以下のフォーマットでまとめる:

```
## Xリサーチ結果

情報ソース: {WebSearch / Nitter / X API}（使用したPhaseを記載）

### トレンド・話題
- {箇条書きで3〜5件}

### 注目の意見・切り口
- {箇条書きで3〜5件}

### 使えそうなフレーズ・表現
- {箇条書きで3〜5件}

### 高エンゲージメント投稿（X API使用時のみ）
- {いいね・RT数が多い投稿の要約}
```

  header: "リサーチ確認"

- **背景・口癖・トーン参考**: 残りのRAG情報は、冒頭の問題提起や口調の参考として使う（事例として明示しない）

- Shorts②: How-to型（60秒）

### 台本生成時の重要ルール

1. **RAGデータ活用**: ステップ④で抽出した山本さんの具体的な数字・エピソード・実体験を必ず織り込む
2. **話し言葉**: すべて話し言葉。「〜ですね」「〜じゃないですか」「めちゃめちゃ」「ぶっちゃけ」等のBRAND DNA口調を使う
3. **数字の裏付け**: 具体的な数字を最低8箇所に含める（ただし具体事例としての使用はコア事例に限定し、数字だけの引用は「背景情報」枠から活用する）
